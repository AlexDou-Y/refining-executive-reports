# 高管汇报材料润色 Skill

Skill 名称：`refining-executive-reports`

## 一、这是什么

这是一个面向 CEO、总裁、经营管理层及其他高管的中文汇报材料润色 Skill。

它不仅修改句子，还会从汇报对象的阅读视角出发，优化材料的主线、结构、标题、数据、成果和价值表达，帮助将“事项罗列型文本”转化为结论清晰、重点突出、表达专业克制的高管汇报材料。

适用于半年度／年度工作简报、季度汇报、专项总结、项目复盘、经营分析、组织与职能工作汇报等场景。

## 二、重点规则

- 从事项罗列转向成果、机制、能力和价值表达。
- 减少过程细节、空泛口号、弱化语气和口语化表达。
- 数据体现基准、变化和趋势，避免不同维度简单并列。
- 标题直接体现关键成果或价值，但避免公式化。
- 尊重职能定位，HR场景优先使用“运营支撑、组织赋能、员工保障”等表达，避免泛化使用“人员管理”。
- 不虚构事实，不拔高阶段性成果，不将“初步形成”写成“全面建成”。

## 三、适合解决的问题

- 原稿过长、细节过多，不适合高管快速阅读。
- 内容停留在“做了什么”，没有讲清成果、变化和价值。
- 标题只是事项名称，无法体现重点。
- 数据缺少同比、环比、基准或趋势判断。
- 文字存在“基本保障、持续强化、取得良好成效”等弱化或空泛表达。
- HR等职能汇报过多使用“人员管理”，没有体现运营支撑、员工保障和组织价值。
- AI、知识库、机制建设等内容只写“上线了什么”，没有说明形成了什么能力。

## 四、如何使用

### 方式一：显式调用

在提示词中写出 Skill 名称：

```text
使用 $refining-executive-reports，将下面内容改成面向 CEO 的汇报材料，控制在 300 字以内，不新增事实：

[粘贴原始内容]
```

显式调用最直接，适合需要稳定使用该写作风格的场景。

### 方式二：使用自然语言提出要求

也可以直接说明汇报对象和修改目标，例如：

```text
请把下面内容改成面向总裁的半年度工作简报，突出成果和业务价值，减少过程细节，不新增事实。
```

## 五、推荐提供的信息

为了获得更准确的结果，建议尽量说明：

- 汇报对象：CEO、总裁、业务负责人或经营管理层。
- 材料类型：摘要、半年度简报、专项汇报、项目复盘等。
- 修改范围：单段、单个模块或整篇材料。
- 篇幅要求：字数、页数或精简比例。
- 输出偏好：直接给正文，还是先给结构建议。
- 事实边界：必须保留的数据、名称、时间和完成状态。
- 特殊定位：例如HR定位为运营支撑，避免泛化使用“人员管理”。

## 六、常用调用示例

### 1. 修改单段内容

```text
使用 $refining-executive-reports 精简下面这段内容，面向 CEO 汇报，只输出修改后的正文，不超过 150 字，不新增事实：

[粘贴原文]
```

### 2. 优化整篇材料

```text
使用 $refining-executive-reports 优化这份半年度工作简报。先用不超过三点说明整体修改方向，再输出完整高管版正文。保留原有数据，不写下一阶段计划。
```

### 3. 优化数据表达

```text
使用 $refining-executive-reports 改写以下数据段落。补充可直接计算的同比变化和趋势判断，将不同维度的数据分组表达，不新增指标。
```

### 4. 优化HR汇报

```text
使用 $refining-executive-reports 将以下HR工作总结改成面向总裁的汇报材料。突出人才供给、运营支撑、员工保障和组织价值，避免泛化使用“人员管理”。
```

### 5. 只要最终版本

```text
使用 $refining-executive-reports 直接输出优化后的最终稿，不需要文本诊断、修改说明或“业务价值”等机械分段。
```

## 七、默认输出方式

- 修改单段或用户要求“直接润色”时：默认只输出修改后的正文。
- 优化整篇或结构混乱时：默认先给不超过三点修改方向，再输出完整正文。
- 不机械增加“总体判断、工作成效、业务价值、修改说明”等栏目。
- 不保留没有具体行动、责任、期限或风险的泛化计划。

## 八、事实与表达边界

本 Skill 不会：

- 虚构数据、成果、政策或业务事实。
- 将试点、拟定或在建事项写成已经全面完成。
- 为了表达有力而使用“全面解决、彻底消除、确保无风险”等绝对口径。
- 在没有数据或事实支撑时使用“显著提升、取得重大成效”等结论。
