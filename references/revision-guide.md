# 高管汇报改写参考

## 一、常见问题与处理

| 原文问题 | 修改方法 |
|---|---|
| 事项平铺 | 先归并主线，再按成果重要性排序 |
| 过程过多 | 删除沟通、会议和执行步骤，只保留关键动作 |
| 语气偏弱 | 用已完成事实直接陈述，不用“基本、一定程度上”自我削弱 |
| 口号空泛 | 将“强化、赋能、推进”落到对象、动作和结果 |
| 数据堆叠 | 补充基准、变化和趋势；不同维度分组 |
| 价值生硬 | 将价值自然融入段落末句，不固定设置“业务价值”标签 |
| 过度拔高 | 按事实使用“建成、形成、初步形成、试点、在建” |
| 职能视角过重 | 从员工体验、业务进展和公司能力重新表述 |

## 二、语言校准

优先使用具体动词：完成、建成、上线、打通、建立、明确、统一、补齐、覆盖、落地、沉淀、降低、提升、支撑、保障。

谨慎使用以下表达：

| 避免机械使用 | 原因与替代方向 |
|---|---|
| 总体判断 | 容易显得生硬；按场景使用“摘要、核心结论、进展概述”或直接进入正文 |
| 基本保障了 | 削弱完成度；改写为具体覆盖结果或“为……提供保障” |
| 持续强化／深入推进 | 信息不完整；补充强化了什么、形成了什么结果 |
| 取得良好／显著成效 | 无事实支撑时空泛；直接列成果或数据 |
| 管好团队／不让员工吃亏 | 过于口语；改为“提升团队稳定性、保障员工权益” |
| 打赢战役 | 口号化；改为“提升团队作战效能、支撑业务目标达成” |
| AI化落地／智能赋能 | 未说明能力变化；写清人机双读、调用方式和应用场景 |

保持自信与审慎的平衡：事实已经完成时直接写“完成、建成、上线”；只有确属阶段性状态时使用“初步形成、逐步完善”。不要为了审慎把所有成果都写弱，也不要为了拔高改变事实状态。

## 三、结构与标题

全文优先采用：

1. 摘要：业务背景＋核心主线＋关键成果＋总体价值。
2. 规模或经营概况：当前状态＋同比／环比变化＋结构趋势。
3. 核心成果：按平行主题展开，每个主题突出结果和价值。
4. 风险、问题或下一步：仅在用户要求或确有必要时呈现。

标题可参考“成果＋价值”思路，但不要让全文都呈现同一种句式：

- 打通属地自主雇佣闭环，沉淀可复制经验
- 建立全量保险体系，保障员工安全与业务稳健运行
- 海外HR知识库正式上线，AI Native交付能力落地

## 四、数据表达

数据服务于判断，不只承担陈列功能。当前值与同期值可直接推导时，规模和覆盖变化优先写同比增幅，绝对增量仅作补充；不同基数的指标不要只用“明显增长”笼统概括。

原始数据：海外常驻员工38人、覆盖14个国家和地区；去年同期20人、9个国家和地区。

推荐表达：截至报告期末，海外常驻员工38人，覆盖14个国家和地区；人员规模同比增长约90%，区域覆盖同比增长约56%，海外业务人员配置加速拓展。

若同段还有岗位结构、本地雇佣占比等数据，先写规模变化，再写结构变化；不要把人数、国家数、覆盖率、项目数混成一组“核心指标”。

## 五、价值表达

只提炼与事实直接相关的价值：

- 对员工：权益保障、安全感、发展机会、公平性、稳定性、体验。
- 对业务：人才到位、运营效率、团队作战、市场拓展、交付稳定。
- 对公司：合规风险、成本效率、机制沉淀、组织能力、可复制经验。

避免每段机械写满三个视角。优先写最直接的一个或两个价值，再在摘要中完成整体上提。

## 六、完整示例

原文：我们推动多个部门沟通，完善了外派员工境内外发薪和税社处理，基本解决了员工收入差异问题，有利于大家出海。

改写：建立外派员工境内外薪资拆分、汇差清算及税社平衡机制，切实保障员工权益，提升外派政策的公平性和可执行性，为稳定海外团队、鼓励人才出海及业务长期拓展提供支撑。

该改写删除沟通过程，保留核心机制，并将价值落到员工权益、团队稳定和业务拓展，未使用“彻底解决”等超出事实边界的表述。
